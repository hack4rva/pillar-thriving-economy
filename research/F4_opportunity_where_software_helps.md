# Where Software Genuinely Helps vs. Decorates Problems

## ✅ DO BUILD — Software Solves Real Confusion

| Opportunity | Why It Works |
|---|---|
| **Scattered information in one mind** | Software consolidates what exists but lives across emails, documents, heads. Reduces re-explanation time. |
| **Coordination at scale** | Software synchronizes when humans can't hold all dependencies. Scheduling, task handoffs, dependencies. |
| **Cognitive load on decision points** | Software surfaces what matters when a human needs to decide. Dashboards, alerts, prioritized lists. Decision clarity improves. |
| **Version/audit confusion** | Software tracks "what changed and who did it." Eliminates argument about history. |
| **Latency in feedback loops** | Software shows impact immediately rather than monthly reports. Faster course correction. |
| **Pattern spotting in noise** | Software finds signal humans would miss (anomalies, trends, correlations in large datasets). |
| **Workflow bottlenecks with clear handoffs** | Software sequences dependent steps reliably when the logic is deterministic. |

## ❌ DON'T BUILD — Software Won't Fix These

| Problem | Why Software Fails |
|---|---|
| **Misaligned incentives** | Software can't make people care about the right outcome. A faster system just accelerates the wrong behavior. |
| **Undefined process** | Software formalizes and locks confusion. Automating an unclear workflow embeds it. Fix the process first. |
| **Trust gaps** | Software can't build trust between teams who don't believe each other. It can only expose the conflict faster. |
| **Leadership clarity missing** | Software won't tell people what they should actually be doing. That's a strategy problem, not an information problem. |
| **Authority disputes** | Software can't decide who decides. If ownership is unclear, software just creates new bottlenecks around "who approves in the system?" |
| **Resource scarcity** | Software can't create time or money. Efficiency theater for a fundamentally under-resourced function. |
| **Cultural resistance** | Software adds friction if the org isn't ready to change behavior. The system becomes "the thing people work around." |
| **Rare, novel decisions** | Software can't replace judgment when each situation is unique. It just produces compliance theater. |

---

## Key Test
**Ask:** "If we removed the software, would the problem still exist?" 
- **Yes** → Don't build. Fix the underlying issue first.
- **No** → Proceed. Software genuinely solves it.
