Research Prompt Package for Pillar 4 (Thriving Economy)

Purpose
- Each file in this folder is a standalone prompt intended for use with a research LLM (e.g., Perplexity API) to generate Richmond-specific findings.
- Prompts focus on procurement access for minority-owned businesses and small-business setup/navigation.

Usage
- Run prompts individually. Keep outputs clearly labeled as Verified (with source URLs), Inference, or Unknown.
- Do not invent facts. Prefer official City and partner sources. Note recency.

Contents
- 00_pillar_summary_context.txt (context only)
- 01_master_research_prompt.txt
- 99_research_plan.md (structured plan to run the prompts and evaluate options)
- A1–A5 Problem Landscape
- B1–B5 User and Stakeholder Research
- C1–C5 Existing Services, Programs, and Interventions
- D1–D5 Data and Source Inventory
- E1–E5 Prior Art and Comparable Solutions
- F1–F5 Opportunity Framing for Hackathon Teams
- G1–G5 Risks, Ethics, and Guardrails
- H1–H5 MVP Feasibility
- I1–I5 Demo and Judging Alignment
- 90_top_10_research_questions.txt
- 91_top_10_hypotheses_to_test.txt
- 92_red_flags.txt
- 93_missing_information_gaps.txt

Meta templates (for cloning to other pillars)
- See `05_prompts/meta/pillar_research_prompt_package_meta.txt` and `.yaml` for a reusable generator that creates this entire prompt package structure from a pillar outline.
